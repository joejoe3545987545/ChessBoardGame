# 开发需求文档 — ChessBoardGame

## 项目概述

ChessBoardGame 是一款结合**五子棋**与**卡牌**玩法的策略游戏。玩家在 15×15 棋盘上下棋，同时通过完成三连/四连模式来抽取卡牌，使用卡牌效果改变战局。

## 核心玩法需求

### 1. 棋盘系统
- 15×15 标准五子棋棋盘
- 黑白双方轮流落子
- 五子连线判定胜利
- 三连/四连模式检测 → 触发抽卡

### 2. 行动点系统 (AP)

#### 2.1 行动点定义
```
struct ActionPoint {
    bool canPieceDrop;  // 是否允许下棋
    bool canPlayCard;   // 是否允许出牌
}
```

#### 2.2 三种判定模式
| canPieceDrop | canPlayCard | 行为 |
|:---:|:---:|---|
| true | true | 可下棋 + 可出牌（双能点） |
| true | false | 仅可下棋 |
| false | true | 仅可出牌 |

#### 2.3 行动点流转
1. 每回合开始 → 自动获得 1 个双能点（true, true）
2. 玩家执行行动 → 消耗对应行动点（优先消耗专能点，保留双能点）
3. 卡牌效果可赠送额外行动点
4. 每次行动后总体结算：
   - AP 池为空 → 回合结束，切换玩家，重置倒计时
   - AP 池非空 → 继续行动，倒计时锁定不重置

### 3. 卡牌系统

#### 3.1 抽卡机制
- 玩家落子产生三连或四连 → 自动从牌库抽 1 张卡牌加入手牌

#### 3.2 出牌机制
- 拖拽卡牌到 CardReader 区域 → 湮灭动画 → 触发卡牌效果
- 出牌消耗 1 个"允许出牌"的行动点
- 无出牌权限时拖到 CardReader → 弹回卡槽

#### 3.3 卡牌效果类型
| 效果 | 枚举 | 说明 | 状态 |
|------|------|------|:---:|
| 强制落子 | FORCE_DROP | 赠送额外落子行动点 | ✅ |
| 修改胜利规则 | CHANGE_WIN_RULE | 改变胜利条件（如五子→六子） | ❌ |
| 移除对方棋子 | REMOVE_OPPONENT | 移除棋盘上对方一颗棋子 | ❌ |
| 额外回合 | EXTRA_TURN | 获得完整额外回合 | ❌ |

### 4. 回合与计时
- 每回合 45 秒倒计时
- 回合结束时超时方判负
- 行动点未耗尽前倒计时**不重置**
- 仅在 AP 池归零、回合切换时重置倒计时

### 5. 游戏模式
- **PVP**：双人热座对战
- **PVE**：人机对战（三级 AI 难度：初级/明智/大神）
- **历史复盘**：回放已保存的对局
