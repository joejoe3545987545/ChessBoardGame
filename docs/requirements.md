# 开发需求文档 — ChessBoardGame

## 项目概述

ChessBoardGame 是一款结合**五子棋**与**卡牌**玩法的策略游戏。玩家在 15×15 棋盘上下棋，同时通过完成三连/四连模式来抽取卡牌，使用卡牌效果改变战局。

## 核心玩法需求

### 1. 棋盘系统
- 15×15 标准五子棋棋盘
- 黑白双方轮流落子
- 每方独立胜利条件（默认 5 子连星）
- 三连/四连模式检测 → 触发抽卡

### 2. 行动点系统 (AP)

#### 2.1 行动点定义
```
struct ActionPoint {
    bool canPieceDrop;  // 是否允许下棋
    bool canPlayCard;   // 是否允许出牌
}
```

#### 2.2 三种判定模式
| canPieceDrop | canPlayCard | 行为 |
|:---:|:---:|---|
| true | true | 可下棋 + 可出牌（双能点） |
| true | false | 仅可下棋 |
| false | true | 仅可出牌 |

#### 2.3 行动点流转
1. 每回合开始 → 自动获得 1 个双能点（true, true）
2. 玩家执行行动 → 消耗对应行动点（优先消耗专能点，保留双能点）
3. 卡牌效果可赠送额外行动点
4. 每次行动后总体结算：
   - AP 池为空 → 回合结束，切换玩家，重置倒计时
   - AP 池非空 → 继续行动，倒计时锁定不重置

### 3. 卡牌系统

#### 3.1 抽卡机制
- 玩家落子产生三连或四连 → 自动从牌库抽 1 张卡牌加入手牌
- AI 同样享有抽卡权利（共享牌库）

#### 3.2 出牌机制
- **橙卡**：拖拽到 CardReader 区域 → 边滑入边碎片销毁 → 中央展示 → 触发效果
- **紫卡**：拖拽到 CardPortal 区域 → 下滑传送动画 → 进入敌方手牌
- 出牌消耗 1 个"允许出牌"的行动点
- 无出牌权限时拖到 Reader/Portal → 弹回卡槽

#### 3.3 橙卡效果
| 卡牌 | 效果 | 说明 | 状态 |
|------|------|------|:---:|
| 连击 | FORCE_DROP | 赠送 2 次额外落子机会 | ✅ |
| 隐忍 | CHANGE_WIN_RULE | 敌方胜利条件改为六子连星 | ✅ |
| 笼络 | CONVERT_PIECE | 销毁己方一子 + 转化敌方一子 | ✅ |
| 破釜沉舟 | SACRIFICE_HAND | 手牌回牌库 + 按数量销毁敌方棋子 | ✅ |
| 疫病 | PLAGUE | 感染敌方棋子，每回合概率死亡+传播 | ✅ |
| 隔离 | QUARANTINE | 三回合后清除所有疫病 | ✅ |

#### 3.4 紫卡效果
| 卡牌 | 效果 | 说明 | 状态 |
|------|------|------|:---:|
| 盲目 | BLIND | 手牌名/描述乱码，三回合消退，三连延后 | ✅ |
| 以地事秦 | YIDISHIQIN | 需传橙卡回应，否则三回合禁出牌 | ✅ |

### 4. 碎片动画系统
- 棋子销毁：12×12=144 碎片，自下往上逐行粉碎坠落
- 棋子转化：敌方粉碎 + 我方碎片聚合
- 卡牌销毁：36×36=1296 碎片，边上升边粉碎
- 卡牌出生：碎片从下方飞入聚合，1.5s
- 展示动画：碎片聚合自上往下扫描
- 破釜沉舟退牌：所有手牌同时碎片销毁
- 碎片残留衰减：动画结束后继续播放至自然淡出

### 5. 回合与计时
- 每回合 45 秒倒计时
- 回合结束时超时方判负
- 行动点未耗尽前倒计时**不重置**
- 仅在 AP 池归零、回合切换时重置倒计时

### 6. 游戏模式
- **PVP**：双人热座对战
- **PVE**：人机对战（三级 AI 难度：初级/明智/大神）
- **历史复盘**：回放已保存的对局
- **暂停菜单**：ESC 暂停/恢复

### 7. 疫病系统
- 感染敌方棋子，每回合：敌方 10%/我方 5% 概率销毁
- 存活棋子四向传播（敌方 60%/我方 40%）
- 潜伏期：新感染当回合不传播
- 隔离卡：三回合后清除全部疫病
