# 执行步骤 — ChessBoardGame

## 当前开发阶段：卡牌功能基本完成，进入打磨阶段

## 已完成里程碑

### ✅ 基础棋盘系统
- 15×15 棋盘渲染与落子交互
- 五子/六子连线胜利判定 + 高亮（每方独立 winCondition）
- 三连/四连模式检测 + 自动抽卡
- 落子历史记录与复盘回放

### ✅ 行动点框架
- 三模式判定（双能/仅下棋/仅出牌）
- 智能消耗算法（优先专能点，保留双能点）
- 总体结算中心（AP 归零切回合，未归零继续）
- 倒计时锁定（动画期间冻结）

### ✅ 卡牌系统
- 2 张卡牌：连击（FORCE_DROP，+2 落子点）、隐忍（CHANGE_WIN_RULE，敌方→六子连星）
- 抽卡：三连/四连自动抽取，出生动画（连续插值 + 白光消退）
- 出牌：拖到 CardReader → 5 阶段湮灭 → 中央展示 → 效果触发
- 展示动画：PAUSE 0.3s → APPEAR 1.2s（缓出缩放+白光）→ DISPLAY 1.2s → FADE_OUT 0.4s
- 展示文字：统一缩放因子，等比放大

### ✅ 双卡槽系统
- 槽 1 (1080, 160)、槽 2 (1080, 480)，各有独立堆叠（+5px 偏移）
- 双槽存放：吸附态点击槽区即存入，自动置顶
- 双槽独立拾取：每槽顶卡可被吸附，显式计算包围盒
- 存入即栈顶：存放时自动 reorder 到 hand 末尾

### ✅ 渲染系统
- 统一 `activeIdx`：吸附/湮灭/回弹/默认四种状态统一管理活跃卡
- 精灵位置保存恢复：防静态循环残留坐标污染状态机
- 文字指向活跃卡：不再强制 `hand.back()`
- 每张卡都渲染文字（渲染顺序天然遮挡）

### ✅ 音乐系统
- 动态战斗音乐：NORMAL ↔ BATTLE_02 状态机 + 1s crossfade
- CardReader / 四子连星触发切歌
- 危机叠加延长（无上限）
- 回合计时暂停：动画期间不计时
