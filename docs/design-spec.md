# 设计规范 — ChessBoardGame

## 设计理念

**"魔幻像素"风格** — 结合复古像素美学与卡牌魔幻元素。UI 应具有神秘感和仪式感。

## 视觉设计

### 配色方案
| 用途 | 颜色 | 场景 |
|------|------|------|
| 棋盘底色 | 暖木色 | 棋盘背景 |
| 黑子 | 纯黑 + 微光泽 | 棋子渲染 |
| 白子 | 纯白 + 微光泽 | 棋子渲染 |

### 卡牌交互设计

#### 状态机
```
[插槽常驻] ←→ [鼠标吸附] → [CardReader湮灭] 或 [弹回卡槽]
```

#### 湮灭动画（5 阶段）
1. **原地停顿** — 松开后静止 0.2s
2. **落向中心** — 缓动至 (640, 200)
3. **停顿读取** — 定格 0.5s（模拟读卡）
4. **虚无上升** — 慢速升至 (640, 70)
5. **最终定格** — 停顿 0.5s + 白光褪色 + 文字淡出 → 销毁

总时长：约 1.2s

## 交互设计

### 出牌操作
- **抓取**：单击卡牌 → 吸附到鼠标
- **出牌**：拖到 CardReader 圆形区域内松开 → 湮灭 → 触发效果
- **取消**：在 CardReader 外松开 → 弹回卡槽
- **拦截**：无出牌行动点时拖到 CardReader → 弹回卡槽（拒绝出牌）

### 下棋操作
- 单击棋盘空白交叉点 → 落子
- 无下棋行动点时点击 → 拦截，不落子

### 按钮操作
- 底部三个按钮：认输、重新开局、返回主页
- 按钮有悬停高亮反馈

## 卡牌设计

### 现有卡牌
| ID | 名称 | 效果 | 描述 |
|:---:|------|------|------|
| 1 | 连击 | FORCE_DROP | 给予两次额外落子机会 |
| 2 | 规则加固 | CHANGE_WIN_RULE | 胜利条件从五子变为六子 |

### 设计原则
- 每张卡牌有明确的战术价值
- 效果通过行动点系统间接影响战局（而非直接操作棋盘）
- 避免"万能卡"——每张牌应有取舍
- 卡牌名称和描述使用中文

## 音效设计

| 音效 | 触发条件 |
|------|----------|
| 落子音 | 棋子放置到棋盘时 |
| 背景音乐 | 不同场景切换不同 BGM（菜单/配置/对战/胜利/失败） |

## 字体规范

- UI 字体：通过 SFML `sf::Font` 加载（主系统字体）
- 卡牌文字：独立卡牌专属字体 `cardFont`
- 卡牌名称/描述：在湮灭动画阶段 5 时渐显
